# 데일리 과제 자동화 프로젝트 컨텍스트

## 프로젝트 목적

이 프로젝트는 Notion에 이미 작성된 데일리 과제를 분석하고, 부족한 부분을 찾아 보완하는 에이전트를 만든다.

에이전트는 과제를 처음부터 대신 작성하지 않는다. 사용자가 작성한 내용과 의도를 최대한 보존하면서 구조와 완성도를 높이는 것이 핵심이다.

## 핵심 기능

- 기존 Notion 데일리 과제 내용을 불러오고 구조를 파악한다.
- 누락되었거나 설명이 부족한 부분을 식별한다.
- 원문을 임의로 대체하지 않고 보완안을 제안하거나 반영한다.
- 토글, 소제목, 목차의 순서와 계층을 일관되게 정리한다.
- 변경 전후의 차이와 보완 이유를 사용자가 확인할 수 있게 한다.

## 제품 원칙

1. **보완 우선**: 새 문서를 생성하기보다 기존 문서의 빈틈을 메운다.
2. **작성자 의도 보존**: 원문의 의미, 말투, 결론을 불필요하게 바꾸지 않는다.
3. **근거 있는 변경**: 각 보완 사항은 어떤 기준으로 필요한지 설명할 수 있어야 한다.
4. **최소 변경**: 목적을 달성하는 데 필요한 범위만 수정한다.
5. **검토 가능성**: Notion에 반영하기 전에 변경 내용을 검토할 수 있어야 한다.
6. **구조 일관성**: 토글과 소제목의 계층, 명명 방식, 순서를 문서 전체에서 통일한다.

## 기본 처리 흐름

1. Notion에서 대상 데일리 과제를 읽는다.
2. 본문, 토글, 소제목과 계층 구조를 내부 표현으로 변환한다.
3. 내용의 누락, 불명확성, 구조 문제를 분석한다.
4. 원문 보존 원칙에 따라 보완안과 구조 정리안을 만든다.
5. 변경 사항과 이유를 사용자에게 보여준다.
6. 사용자 승인 후 Notion 문서에 반영한다.
7. 반영 결과를 다시 읽어 의도한 구조인지 확인한다.

## 기술 방향

- API 서버: FastAPI
- 외부 도구 및 Notion 연동: MCP
- 언어와 세부 라이브러리: 구현 단계에서 확정
- 외부 시스템 변경은 읽기와 쓰기를 분리하고, 쓰기 작업은 명시적인 승인 뒤 수행

## 범위

### 포함

- 기존 데일리 과제 분석
- 부족한 내용의 보완
- 토글, 소제목, 목차 구조 정리
- 변경안 미리보기와 승인 기반 반영

### 현재 제외

- 아무 내용도 없는 과제를 처음부터 완성해 주는 기능
- 사용자 확인 없는 원문 전체 재작성
- 데일리 과제와 무관한 범용 Notion 문서 자동 편집

## 협업 규칙

- 구현 전에 이 파일을 읽고 프로젝트 목적과 현재 작업이 일치하는지 확인한다.
- 요구사항이 불명확하면 작성자 의도를 보존하는 쪽으로 가장 작은 가정을 한다.
- 외부 시스템에 쓰기, 삭제 또는 대량 변경을 수행하기 전에는 사용자 승인을 받는다.
- 새 아키텍처 결정은 이후 명령 및 결정 문서에 기록한다.
- 특정 하위 폴더에만 필요한 지식은 루트 컨텍스트를 비대하게 만들지 않고 해당 폴더의 `nested.md`에 둔다.

## 후속 단계에서 정할 사항

- 자주 쓰는 명령, MCP 도구, 아키텍처 결정, 중요 파일 위치를 기록할 문서
- 단위 테스트, 스크린샷 또는 시뮬레이터를 이용한 결과 검증 방식
- 에이전트와 MCP 도구별 읽기·쓰기 권한 정책
- Notion 데일리 과제의 실제 템플릿과 필수 구성 요소
- 보완 필요성을 판정하는 구체적인 기준
